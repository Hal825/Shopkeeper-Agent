MySQL 中文输入问题 & 字段索引缺失排查全过程
一、问题现象
在构建元数据知识库时，执行 build_meta_knowledge.py 脚本，发现 Elasticsearch 中仅写入 15 条字段取值，预期 distinct 非空值总数应为 24 条（各维表字段去重后数量之和），数据存在明显丢失。
日志输出示例
text
表=dim_region 字段=province → 查询结果数量：1
表=dim_region 字段=region_name → 数量：1
...
表=dim_product 字段=product_name → 数量：14
表=dim_product 字段=brand → 数量：2
表=dim_date 字段=quarter → 数量：1
最终准备写入 ES 总数：15   # 仅 15 条，丢失 9 条
数据库实际包含 6 个不同省份、2 种性别、4 个会员等级等数据，但 SQL 查询仅返回 1 条结果。
二、排查过程
2.1 第一步：怀疑 sync 配置未生效
问题定位
_save_value_info_to_es 方法原使用字典 col_sync 映射字段同步状态，大量字段 sync 为 False。
原因：字典 key 拼接方式 f"{table.name}.{col.name}" 与 col_info.id 存在空格、大小写等微小差异，导致字段匹配失败。
解决方法
改为直接遍历 meta_config.tables，实时匹配 full_id == col_info.id，确保同步配置正确获取。
处理结果
所有字段正常进入处理流程，但 province 等字段查询数量仍为 1，问题未解决。
2.2 第二步：检查 get_column_values 查询结果
在 DWMySQLRepository.get_column_values 中添加调试日志，打印 SQL 与原始返回值：
python
运行
sql = f"select distinct {column_name} from {table_name} limit {limit}"
print(f"[SQL] {sql}")
result = await self.session.execute(text(sql))
rows = result.fetchall()

排查结果
province 查询结果为 [('',),]（空字符串），region_name/customer_name/gender/member_level/category 等字段均返回空字符串。
结论：数据库实际数据为空，非代码过滤逻辑导致。

2.4 第四步：确认 SQL 插入语句是否正确
初始化 SQL 插入语句已明确写入中文值，示例：
sql

但执行后中文变为空串；手动在 MySQL 终端执行带中文 INSERT，中文输入 / 粘贴后立即消失，仅剩空引号。
2.5 第五步：定位根本原因 —— 容器 locale 不是 UTF-8
执行命令检查容器 locale：
bash
运行
docker exec -it mysql locale
输出显示 LANG= 或 POSIX，不支持 UTF-8 编码。
核心原理
MySQL 服务器端 character_set_server=utf8mb4 仅保证存储、返回编码正常；
客户端输入的字节解析由操作系统 locale 决定；
容器默认 locale 为 C/POSIX，不支持中文多字节字符，输入时中文被丢弃 / 替换为空。
三、解决方案
3.1 修改 docker-compose.yml，添加 UTF-8 环境变量
yaml
services:
  mysql:
    image: mysql:8.0
    container_name: mysql
    environment:
      MYSQL_ROOT_PASSWORD: "@Lzs0825"
      MYSQL_USER: lzs
      MYSQL_PASSWORD: Lzs666
      LANG: C.UTF-8          # 关键：强制容器使用 UTF-8 locale
    ports:
      - 3306:3306
    volumes:
      - mysql_data:/var/lib/mysql
      - ./mysql:/docker-entrypoint-initdb.d
    command:
      --character-set-server=utf8mb4
      --collation-server=utf8mb4_general_ci
      --skip-character-set-client-handshake   # 忽略客户端字符集，强制服务端 utf8mb4
3.2 重建容器（数据保留）
bash
运行

docker-compose up -d

四、涉及的关键代码调整
4.1 _save_value_info_to_es 改进（sync 匹配优化）
原字典映射不可靠，改为遍历配置匹配：
python
运行
sync = False
for table in meta_config.tables:
    for col in table.columns:
        if f"{table.name}.{col.name}" == col_info.id:
            sync = col.sync
            break
    if sync:
        break
4.2 空值过滤增强
过滤空字符串与 None，避免无效数据写入 ES：
python
运行
if val is None or (isinstance(val, str) and val.strip() == ''):
    continue
4.3 ES 批量写入
使用 bulk API 分批写入，避免单条数据覆盖问题。
五、经验总结
表格
问题现象	初步怀疑原因	最终根因
字段查询返回 1 条	sync 配置未生效	字典 key 不匹配 → 改为直接遍历匹配
查询返回空字符串	代码逻辑过滤数据	数据库存储本身为空
插入中文变空串	MySQL 字符集配置错误	容器 locale 不支持 UTF-8
服务端 charset=utf8mb4 仍无法输入中文	数据库配置失效	locale 决定输入解析，与服务端无关
核心教训
Docker 容器运行数据库时，需同时配置两项关键设置：
容器环境：LANG: C.UTF-8（支持中文编码）；
MySQL 服务端：指定 --character-set-server=utf8mb4 + --skip-character-set-client-handshake。

功能验证：支持中文搜索，如查询「黄金会员」可正常返回 dim_customer.member_level.黄金。
